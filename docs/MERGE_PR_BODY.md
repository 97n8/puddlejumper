## What

Merge the standalone `97n8/LogicOS` repo into puddlejumper as `apps/workspace`. Retire the LogicOS brand in code. The standalone repo stays live for 30 days post-merge, then gets archived.

## Why

LogicOS was always the PJ operator frontend, despite living in its own repo. Maintaining two repos meant:

- Type drift between API and frontend (no shared `packages/types`)
- Two CIs to keep green, two PR queues to babysit
- Two Vercel projects with separately-managed env vars
- Brand confusion — PJ canon says LogicOS is retired; the repo said otherwise

One source of truth, one PR queue, one deploy story.

## What moved

- **Deleted:** `apps/logicos/` — a 50-file mobile-shell stub imported via PR #96, not the real LogicOS
- **Added:** `apps/workspace/` via `git subtree add --squash` from `97n8/LogicOS@main`
- **Renamed package:** `@publiclogic/workspace`

## Brand changes inside `apps/workspace/`

- `LogicOS` → `Workspace` (in prose, comments, UI strings)
- `logicos` → `workspace` (code identifiers, CSS classes, import paths)
- `logic-os` → `workspace` (URL slugs)

### Preserved (not renamed)

These are feature names inside the workspace, not the workspace itself:

- `LogicCommons`
- `LogicDocs`
- `LogicBackend`

The rename was applied via `scripts/merge-rename-workspace.sh` with explicit carve-outs for the above. See script source for the exact regex protection.

## Deploy

**Vercel project cutover is NOT part of this PR.** Nate handles separately after merge.

Until cutover:

- The standalone `97n8/LogicOS` Vercel project stays pointed at `97n8/LogicOS` and continues to serve production
- The new `apps/workspace` in puddlejumper has no production deploy target yet
- Preview deploys on PRs in puddlejumper may pick up the workspace depending on monorepo config — check before assuming

**Rollback path:** `git revert -m 1 <merge-commit-sha>` and push. No Vercel changes needed (cutover hasn't happened).
