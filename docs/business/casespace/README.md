# Michigan LTC Network — Resource Recovery Corridor · CaseSpace

**Original files stay preserved. The VAULT Workbook controls the record. Deliverables come from the record.**

This is the corridor's operating record — not a grant dump. Every claim, number, site description, role, grant answer,
public statement, and deliverable traces back to a **source file**, a **workbook row**, and a **final output**. Anyone
entering the project should be able to understand what happened, what is true, what is pending, and what can safely be
reused.

## The control surface
`00_CONTROL/Michigan_LTC_VAULT_Workbook.xlsx` — 16 tabs + Dashboard, pre-populated with the reconciled record as of

## File naming
- **Source files:** `YYYY-MM-DD - Source - Description - Version.ext` (e.g., `2026-05-27 - WasteWerx - Staffing Proforma - Original.xlsx`)
- **Deliverables:** `YYYY-MM-DD - Michigan LTC - Deliverable Name - Audience - Status.ext`
- Never use `final.docx`, `new model.xlsx`, `staffing revised 2.xlsx` — they destroy memory.

## The clean internal standard — every important item has five things
1. **Source file** · 2. **Workbook row** · 3. **Decision or status** · 4. **Owner** · 5. **Output or next action**

> Example — *Staffing Proforma*: Source = WasteWerx Staffing Proforma (SRC-005) · Workbook = OPS rows / FIN-015 ·
> Status = needs Michigan rates + cross-training · Owner = Robert/Vinny/Glen · Output = Staffing Reconciliation Brief.

## Rules
- Originals in `01_SOURCE_FILES` are **never edited** — working versions live elsewhere.
- Removed sites (Coldwater) stay in the Site Register marked **inactive** — never deleted.
- Public language is **source-backed and verification-ready** — no overclaiming (`fully permitted`, `zero-emission`,
  guaranteed returns, unqualified $30.2M EBITDA). See VAULT tabs 10 (Claims) and 12 (Public Language).
- Anything investor-facing waits on securities counsel; PublicLogic is governance, not a placement agent.

*Empty folders carry a `.gitkeep` placeholder. Not financial/legal/securities advice.*
