# Delivery Playbooks

## Standard Engagement Phases
1. Listen and Map (Week 1-2)
   - Interviews with staff and leadership
   - Current workflow capture
   - Compliance requirements mapping
   - Output: Baseline map and risks

2. Design and Document (Week 3-6)
   - Target-state workflow design
   - Plain-language documentation
   - Records retention alignment
   - Output: Draft continuity package

3. Pilot and Iterate (Week 7-10)
   - Run workflows with real cases
   - Gather feedback and adjust
   - Output: Finalized package

4. Transfer and Embed (Week 11-12)
   - Training sessions
   - Handoff materials
   - Output: Adoption plan and next steps

## VAULT Playbook (Continuity and Records)
- Inputs: current retention schedules, records inventory, access policies
- Steps:
  - Map record types to retention needs
  - Define access and request workflows
  - Create continuity binder (roles, steps, checklists)
- Outputs:
  - VAULT package
  - Training session notes
  - Audit checklist

## ARCHIEVE Playbook (Workflow + Governance)
- Inputs: board/committee processes, statutory requirements
- Steps:
  - Map governance workflows (meetings, agendas, minutes, approvals)
  - Align documentation with public access needs
  - Create continuity and compliance steps
- Outputs:
  - ARCHIEVE package
  - Governance workflow map
  - Public-facing documentation pack

## Pilot Playbook
- Define scope and success metrics
- One department or function only
- Weekly check-ins
- Close with continuity package and training

## Quality Checklist
- Compliance mapping documented
- Steps written in plain language
- Handoff materials included
- Audit trail and update cadence defined
