LogicOS User Manual

The complete guide to running your municipality on LogicOS — from signing in to sealing a record before a Select Board meeting.

Your First 15 Minutes in LogicOS

This walkthrough follows a Town Clerk setting up LogicOS for the first time. If you're a Finance Director, DPW head, or Town Administrator — the same path applies, just pick your module in step 4.

Min 0–2

Sign in with GitHub

Go to os.publiclogic.org → click Sign in with GitHub. You'll be back in LogicOS within 30 seconds. Your name and photo appear in the top toolbar.

Min 2–4

Connect Microsoft 365 or Google (optional but recommended)

Click Connect in the toolbar → pick your service → authorize the popup. This lets LogicOS file records directly to SharePoint or Google Drive and send notifications.

Min 4–6

Open the Civic Environment

Click Civic in the toolbar (or go to /civic). Select your town from the list. You'll land in the Civic Setup wizard.

Min 6–10

Configure Org Manager first

Before anything else, open Org Manager (/org) and add your key positions: Town Administrator, Finance Director, Town Clerk, Purchasing Agent. Map the 7 governance roles to people. Every other module resolves its approvers from Org Manager — this is the foundation.

Min 10–13

Configure one module

Return to Civic Setup and configure Board Compliance (quickest) or Records Management. Set the Responsible Officer, pick your document routing, confirm the retention period. That's it — the module is live.

Min 13–15

Seal your first record

Open Vault (/vault) → upload any document → assign a classification → click Seal Record. LogicOS assigns a FormKey, computes a SHA-256 hash, and starts the retention clock. The record is now tamper-evident and defense-ready.

✅ You're live After those 15 minutes you have a configured governance module and at least one verified record. The Watch Layer will begin surfacing compliance items as you add more data.

What is LogicOS?

LogicOS is the PublicLogic operational platform for regulated organizations — built on the principle that governance, compliance, and records must survive personnel changes, audits, and undocumented process. The first deployed environment is Civic, purpose-built for New England municipalities. Additional environments for healthcare organizations, grant-funded nonprofits, and multi-jurisdictional authorities are in development and follow the same architecture.

LogicOS replaces the spreadsheets, email threads, and disconnected software that most organizations use for finance, records, procurement, and board management with a single system that enforces statutory and regulatory obligations automatically.

LogicOS runs at os.publiclogic.org. The backend API is at api.publiclogic.org.

→

Every record is hashed with SHA-256 at the moment of creation. The SEAL system runs server-side and can verify any record's integrity at any time. This is the foundation of LogicOS's tamper-evidence story.

Civic Governance Environment ● Live

The Civic Environment is the full municipal governance desk — one screen that replaces a dozen separate tools. It is MGL-compliant by design: retention periods, approval thresholds, and notice requirements are pre-configured per statute.

⚖ MGL compliance is built in You set the people. LogicOS enforces the law. Every module is pre-wired with the relevant Massachusetts General Law statute, so you can't forget a 48-hour posting window or a $10,000 bid threshold — the system catches it first.

Eligibility — V1 (Current)

Civic V1 serves Massachusetts municipalities with a population under 15,000. This is an intentional V1 constraint — the module configuration, approval chains, and template library are tuned for small-town governance structures (one Finance Director, one Town Clerk, Select Board as the top approval body).

Towns in the 15,000–25,000 range are planned for Civic V2. If your town is near that boundary, email info@publiclogic.org — we may be able to configure V1 for you manually while V2 is in development.

Org Manager ● Live

Org Manager is the structural foundation of LogicOS. It maps your organization — departments, positions, and the people holding them. Every other module resolves its approvers, responsible officers, and escalation targets from Org Manager at runtime.

⚠ Configure Org Manager first If Org Manager is empty, other modules cannot route or escalate. Set it up before running Civic Setup for Finance, Procurement, or CGM. CGM specifically will not function without it.

Vault ● Live

Vault is the governed document store. Every record saved in Vault is immutable — it receives a FormKey, a SHA-256 hash, and a retention tag at creation. The SEAL system verifies the hash server-side at any time, producing proof that the record has not been altered since it was filed.

Uploading and sealing a record

1

Open Vault at /vault

Your governed file library appears. Search, filter by classification, or browse by date.

2

Click Upload or drag a file into the panel

Vault accepts PDF, Word, Excel, images, and plain text. File size limit: 100MB.

3

Assign a classification

Choose from: Invoice, Contract, Meeting Minutes, Public Notice, Bid Document, Personnel Record, Evidence, Other.

4

Seal the record

Click Seal Record. LogicOS assigns a FormKey, computes and stores the SHA-256 hash, and starts the retention clock. The record is now tamper-evident.

Verifying a record

To verify a record hasn't been altered: open the record in Vault → click Verify Seal. The system recomputes the hash and compares it to the stored value. You'll see either a green "Record verified — hash matches" or a red warning.

FormKey ● Live

FormKey is LogicOS's document identification system. Every governed record receives a unique, human-readable key at the moment it is created. The key encodes what the record is, when it was created, and its sequence number.

Pattern: TOWN_MODULE_DOCTYPE_YYYY-MM-DD_NNN_V1

Example: SUTTON_FISCAL_INVOICE_YYYY-MM-DD_001_V1

Why FormKey matters

  • Every document is findable by ID across all modules — no folder hunting
  • The key is human-readable by design: a Town Clerk can read it aloud at a meeting
  • Cross-module linking: an invoice's FormKey connects to its procurement record, contract, and board vote — one click for the full audit trail
  • Defense-ready: in a public records request or legal proceeding, you can produce any record in under 30 seconds by FormKey

The FormKey generator and demo is at /formkey.

Records ● Live

The Records module is the official archive. All sealed records from every module flow here. Records are organized by classification, retention schedule, module origin, and status.

What you can do in Records

  • Search — Full-text search and FormKey lookup across all records
  • Filter — By module, classification, date range, retention tier, or status
  • Export — Download one or multiple records (PDF, original format) for FOIA responses or audits
  • Verify — Run SEAL verification to confirm a record's integrity
  • Transfer — Move records to permanent archival storage when their active period ends

Public Records Requests (Ch. 66)

When a public records request arrives in Intake, it is automatically routed to Records. MGL Ch. 66 §10 requires an initial response within 10 business days of receipt; extensions are available under the statute for requests that require additional time to fulfill. LogicOS starts the clock at Intake, displays a countdown on the item, and escalates to the Records Authority at day 8 if no response has been sent.

Board Compliance ● Live

Board Compliance enforces Massachusetts Open Meeting Law (MGL Ch. 39, §23B) — the statute that governs public notice requirements for all government board meetings.

Open Meeting Law requirements

  • Meeting notices must be posted 48 hours in advance (excluding Sundays and legal holidays)
  • Agendas must specify all topics to be discussed — no "other business" catch-all items
  • Minutes must be approved and made available in a reasonable time
  • Executive sessions require a specific statutory purpose, a recorded vote to enter, and a statement that the discussion will remain confidential

What LogicOS does automatically

  • Sends posting reminders 72 hours before the required 48-hour window closes
  • Flags meetings that have no agenda posted
  • Tracks quorum attendance and alerts when quorum is at risk
  • Auto-archives approved minutes with SEAL verification and a retention tag
  • Generates a compliance report for the Town Clerk showing any Open Meeting Law gaps in the past 12 months

Watch Layer ● Live

The Watch Layer monitors all active modules for compliance health. It surfaces overdue items, escalation triggers, and threshold breaches before they become violations — the difference between "we caught it" and "we received a complaint."

What Watch Layer monitors

  • Invoices nearing the 30-day statutory payment expectation (MGL Ch. 41, §52)
  • Procurement items approaching bid thresholds without the required quotes
  • Meeting notices not yet posted within the 48-hour window
  • Records without retention tags or classification
  • Approval chains pending for more than a configurable number of days
  • Disbursements awaiting second approval past deadline

Access at /watch.

Data Ownership & Export

Your records belong to your municipality. LogicOS is a custodian, not an owner. Here's what that means practically:

Exporting your data

  • Individual records: Open any record in Vault or Records → click Export → download in original format or PDF with the SEAL verification certificate appended.
  • Bulk export: From Records, use the filter to select a date range or classification → click Export All. You'll receive a ZIP file of all matching records.
  • Full account export: Email info@publiclogic.org to request a complete data export. We will provide a structured export within 5 business days.

If you leave LogicOS

Your records are your town's property. On cancellation:

  • You have 30 days to export all records before the account is closed
  • All SEAL verification certificates remain valid after export — you can verify record integrity independently of LogicOS
  • Records filed to SharePoint or Google Drive during your subscription remain in your SharePoint/Drive; they are not deleted when you leave LogicOS
⚖ This is by design VAULT's core principle is that records are the town's property, not the software vendor's. The SEAL hash is computed using open standards (SHA-256) — any compliant tool can verify a LogicOS record without needing LogicOS.

Offline & Outage Behavior

LogicOS is a cloud-hosted application and requires an internet connection for full functionality. Here's what happens in each scenario:

ScenarioWhat you can doWhat you can't do

LogicOS scheduled maintenanceRecords filed to SharePoint or Google Drive remain accessible via those services directlyAccess LogicOS modules

Recommended workaround for meetings

Before any critical meeting, open Board Compliance and print or PDF the agenda and attendance roster. If connectivity is lost mid-meeting, minutes can be entered after connectivity is restored — the SEAL timestamp will reflect when the record was actually sealed, not backdated.

Troubleshooting & FAQ

I signed in with GitHub but Microsoft 365 won't authorize

This is the most common connection issue. The most likely cause is that your Microsoft tenant administrator hasn't approved the PublicLogic application in Azure Active Directory.

  1. When you see the "Approval Required" or "Need Admin Approval" screen in the Microsoft popup, note the application ID and send it to your IT administrator.
  2. The IT administrator can approve it in Azure AD → Enterprise Applications → Admin Consent.
  3. Once approved, try connecting again from the LogicOS Connected Services panel.

If your organization doesn't have a Microsoft 365 tenant administrator, use Google Workspace or email info@publiclogic.org for alternatives.

My Google Drive connection fails to save files

Check that the Google account you connected is the one that has write access to the destination folder configured in Civic Setup. If you connected a personal Gmail but your Drive is a Workspace account, disconnect and reconnect with the correct account.

I can't see the Platform tab in the toolbar

The Platform tab is visible to Admins and Owners only. If you are a Member or Viewer, you won't see it. Ask your workspace Owner to change your role if you need access.

A record shows "Hash mismatch" during SEAL verification

This is a serious integrity flag. It means the stored file content does not match the hash recorded at sealing — the record may have been altered after it was sealed. Do not alter or delete the record. Email info@publiclogic.org immediately and note the FormKey of the affected record.

I accidentally archived a record that shouldn't have been archived

Sealed records cannot be deleted (by design — this is a compliance requirement). Contact support with the FormKey and we can add a correction record linked to the original. The original remains in the archive for the statutory retention period.

The Civic Setup wizard asks for a town I can't find

LogicOS uses Massachusetts DOR municipality codes. If your town is not appearing, it may be because your population is above the V1 threshold of 15,000. Email info@publiclogic.org with your town name — we can often accommodate towns near the boundary manually.

The Automations (Flows) tab is blocked on my phone

This is intentional. The Automations builder requires a desktop browser due to its interface complexity. Open LogicOS on a laptop or desktop computer to create or edit automation rules.

TermDefinition
Acting Start DateThe date a person begins an acting appointment to a position. Tracked separately from permanent appointment for civil service compliance.
ARCHIEVEThe append-only event log that records every action taken on every record — who did what, when. The spelling is intentional (it is a product name, not a typo of "archive"). ARCHIEVE only appends — it never modifies or deletes entries.
Case SpaceTechnical name for a Workspace — a governed collaboration environment. The UI says "Workspace"; the URL uses /casespaces. These mean the same thing.
CGMCapital & Grant Management — the financial governance module for capital projects and grant awards. Requires Org Manager to function.
Ch. 30BMGL Chapter 30B — the Massachusetts statute governing municipal procurement thresholds and competitive bidding requirements.
Compliance Confidence ScoreA 0–100% rating for each module reflecting on-time processing, record completeness, and the absence of overdue flags. Visible in LogicDASH.
DOR CodeMassachusetts Department of Revenue municipality classification code — used to identify towns uniquely in LogicOS.
FormKeyUnique, human-readable document identifier assigned to every governed record. Pattern: TOWN_MODULE_DOCTYPE_YYYY-MM-DD_NNN_V1. Example: SUTTON_FISCAL_INVOICE_YYYY-MM-DD_001_V1.
Governance RoleOne of 7 statutory authority roles in Org Manager (Administrator, Finance Authority, Grant Manager, Project Owner, Auditor, Procurement Authority, Records Authority). Modules resolve approvers from these roles at runtime.
IntakeThe front door for all incoming items — invoices, applications, requests, and correspondence. Intake logs, classifies, and routes everything before it reaches a module.
Live QueueThe real-time list of items moving through a module's process stages (New → Awaiting Approval → In Review → Completed).
MGLMassachusetts General Laws — the statutes governing town operations. LogicOS pre-wires every Civic module with the relevant MGL chapter and section.
One Process, TheThe 8-stage governance flow that every action in LogicOS follows: Receives → Opens → Works → Decides → Records → Notifies → Archives → Learns.
Open Meeting LawMGL Ch. 39, §23B — requires public notice of government meetings and public access to minutes. Board Compliance enforces this automatically.
PuddleJumperThe LogicOS backend API, running at api.publiclogic.org. All data flows through PuddleJumper; the browser never touches provider tokens directly.
SEALThe server-side SHA-256 hash verification system. SEAL proves a record was not altered after it was created. Runs entirely server-side — the browser never sees or writes hashes.
Setup WizardThe guided configuration flow that runs the first time you enter the Civic Environment for a town. Configures each module's Responsible Officer, document routing, retention period, and automations.
VAULTThe governed document store. Every record in VAULT is immutable after sealing, hash-verified, and tagged with a retention period.
Watch LayerThe compliance monitoring system that surfaces overdue items, approaching deadlines, and threshold violations across all active modules.
